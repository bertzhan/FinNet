## 📋 项目概述

本系统实现了美国SEC上市公司（NASDAQ & NYSE）年报数据的**自动化采集与处理**，是一个生产级别的数据工程项目。

### 核心数据指标
- **覆盖公司**: 4,200家（NASDAQ 2,598 + NYSE 1,552）
- **采集文件**: 107,694个文件（10-K年报 + 10-Q季报 + 图片）
- **数据规模**: 122GB
- **时间范围**: 2023-2025财年
- **数据完整性**: **100%**（数据库与文件系统完全匹配）

### 核心技术
- **NLP文本处理**: BeautifulSoup + 编码保持算法（解决乱码问题）
- **并发优化**: 8线程并发下载，性能提升4.4倍（116→512文件/分钟）
- **智能识别**: 外国公司多信号融合识别算法
- **数据工程**: PostgreSQL + SQLAlchemy ORM + ETL流程设计

## 🚀 快速开始

### 前置要求
- Python 3.10+
- PostgreSQL 14+
- 500GB+ 存储空间（完整数据集）

### 安装步骤

```bash
# 1. 解压代码包
unzip SEC-Filings-ETL-导师版.zip
cd SEC-Filings-ETL-Clean

# 2. 创建虚拟环境
python3 -m venv venv
source venv/bin/activate  # Mac/Linux
# 或 venv\Scripts\activate  # Windows

# 3. 安装依赖
pip install -r requirements.txt

# 4. 配置环境变量（重要！）
cp .env.example .env
# 编辑.env，设置SEC_USER_AGENT（必须包含联系邮箱）
nano .env

# 5. 启动数据库（Docker方式）
docker-compose up -d postgres

# 6. 初始化数据库
python main.py init-db

# 7. 运行测试（可选）
pytest tests/ -v
```

### 快速测试

```bash
# 测试1: 构建公司列表（约30秒）
python main.py listings

# 测试2: 小规模回填（10家公司，约1分钟）
python main.py backfill --limit 10

# 测试3: 增量更新（约5-10分钟）
python main.py incremental
```

## 💡 技术亮点

### 1. NLP文本处理
- **编码保持算法**: 解决SEC文档多编码问题（ASCII/UTF-8/Latin-1）
- **HTML解析**: BeautifulSoup + lxml，处理44,263份财报
- **图片提取**: 正则表达式 + DOM树遍历，提取50,898张图片

- **关键技术**: Session-per-thread模式 + 全局速率限制器

### 3. 智能算法
- **外国公司识别**: 多信号融合（表单类型 + 注册地 + 历史文件）
- **去重检测**: SHA-256内容哈希
- **自适应重试**: 指数退避 + 429响应动态调整

### 4. 数据质量保证
- **数据完整性**: 100%（数据库-文件系统一致性验证）
- **下载成功率**: 89.4%（经过自动重试后>95%）
- **SEC合规**: 严格遵守10请求/秒限速

## 📊 数据库架构

### 核心表结构
```sql
companies (4,200条)
  ├── id, ticker, cik, company_name, exchange
  └── is_foreign, country_code  -- 外国公司标识

filings (44,126条)
  ├── accession_number (唯一键)
  ├── form_type (10-K, 10-Q, 10-K/A, 10-Q/A)
  ├── fiscal_year, fiscal_period
  └── is_amendment, amends_accession  -- 修订版本追踪

artifacts (107,694条)
  ├── local_path (存储路径)
  ├── sha256 (内容哈希，用于去重)
  ├── status (downloaded/pending/failed)
  └── retry_count  -- 重试次数

execution_runs (审计日志)
  ├── run_type (listings/backfill/incremental)
  ├── duration_seconds, filings_discovered
  └── artifacts_succeeded/failed

incremental_updates (SLA监控)
  ├── lookback_start/end (时间窗口)
  ├── success_rate, sla_met
  └── new_filings_found
```

## 🔧 常见问题

### Q1: 为什么需要设置SEC_USER_AGENT？
**A**: SEC要求所有API请求必须包含联系信息的User-Agent，否则会被拒绝（403错误）。

### Q2: 数据存储在哪里？
**A**: 默认存储在`/data/filings/`，格式为：
```
/data/filings/NASDAQ/AAPL/2023/AAPL_2023_Q1_03-02-2023.html
/data/filings/NYSE/JPM/2024/JPM_2024_FY_31-01-2025.html
```

### Q3: 如何查看系统运行状态？
**A**: 连接PostgreSQL数据库，查询关键表：
```sql
-- 查看执行历史
SELECT * FROM execution_runs ORDER BY started_at DESC LIMIT 5;

-- 查看下载统计
SELECT status, COUNT(*) FROM artifacts GROUP BY status;

-- 查看SLA指标
SELECT success_rate, sla_met FROM incremental_updates ORDER BY created_at DESC;
```

### Q4: 系统能处理多大规模的数据？
**A**: 当前配置：
- 已测试: 4,200家公司，107,694个文件，122GB
- 理论上限: 可扩展至所有13,000家SEC公司
- 云部署: 支持迁移到S3，可处理PB级数据
