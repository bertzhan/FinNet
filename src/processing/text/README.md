# Chunk - 基于规则的文档结构识别与分块工具

一个基于规则的文档标题层级识别和智能分块工具，专门用于处理中文文档（如年度报告、技术文档等）的结构化分析和分块处理。

## 📋 项目简介

本项目通过**基于规则的标题层级识别算法**，自动识别文档中的标题层级关系，并据此对文档进行智能分块。核心特点是完全基于规则，无需训练数据，适用于中文文档的结构化处理。

## 🎯 核心规则说明

本项目的核心是基于规则的标题层级识别算法。算法通过分析标题的格式和上下文关系，自动识别文档中标题的层级关系。

### 一、标题格式识别规则

算法能够识别以下标题格式：

#### 一级标题
- **格式**: `第X节`、`第X章` 等
- **示例**: `第一节 重要提示`、`第五节 环境和社会责任`
- **特点**: 格式提取为 `None`（需要特殊处理）

#### 二级标题
- **格式**: `X、`（中文数字 + 顿号）
- **示例**: `一、公司信息`、`二、报告期内公司从事的主要业务`
- **格式类型**: `chinese`

#### 三级标题
- **格式1**: `X、`（阿拉伯数字 + 顿号）
  - **示例**: `1、主要业务`、`2、收入与成本`
  - **格式类型**: `arabic`
- **格式2**: `X．`（阿拉伯数字 + 全角句号）
  - **示例**: `1．同一控制下的企业合并`
  - **格式类型**: `arabic_period`
- **格式3**: `X.`（阿拉伯数字 + 英文句号）
  - **示例**: `1. 测试标题`
  - **格式类型**: `arabic_dot`
- **格式4**: `（X）`（中文括号 + 中文数字）
  - **示例**: `（一）行业情况`
  - **格式类型**: `bracket_chinese`

#### 四级标题
- **格式1**: `（X）`（中文括号 + 阿拉伯数字）
  - **示例**: `（1）软件开发业务`、`（2）系统集成业务`
  - **格式类型**: `bracket_arabic`
- **格式2**: `(X)`（英文括号 + 阿拉伯数字）
  - **示例**: `(1) 测试标题`
  - **格式类型**: `bracket_arabic`
- **格式3**: `X）`（只有右括号 + 阿拉伯数字）
  - **示例**: `1）一般处理方法`
  - **格式类型**: `right_bracket_arabic`

#### 五级标题
- **格式**: LaTeX圆圈数字
  - **示例**: `$\textcircled{1}$ 渠道及渠道整合领域`
  - **格式类型**: `latex_circled`

**注意**: 算法会区分细微的格式差异（如空格、括号类型、标点符号），确保只有真正相同格式的标题才会被认为是同级。

### 二、层级调整核心规则

#### 规则1：格式相同且数字连续 → 保持同级 ⭐最高优先级

**描述**: 如果当前标题的格式与前一个标题的格式相同，且数字连续，则当前标题与前一个标题同级。

**条件**:
- 前一个标题格式不为 `None`
- 当前标题格式不为 `None`
- 格式类型相同：`current_format_type == prev_format_type`
- 数字连续：`current_num == prev_num + 1`

**结果**: `adjusted_level = prev_level`

**优先级**: **最高**（规则1优先级高于其他规则）

**示例**:
```
[L4] （1）软件开发业务
[L4] （2）系统集成业务  ← 格式相同（bracket_arabic），数字连续（1→2），保持同级L4
[L4] （3）专业技术服务  ← 格式相同且数字连续（2→3）
```

#### 规则2：从1开始 → 次级标题

**描述**: 无论格式是否变化，只要当前标题从1开始，则当前标题是前一个标题的子标题。

**条件**:
- 当前标题格式不为 `None`
- 当前标题数字为1：`current_num == 1`
- 规则1未应用（如果规则1应用，会覆盖规则2）

**结果**: `adjusted_level = prev_level + 1`

**示例**:
```
[L4] （1）软件开发业务
[L5] 1、核心客户  ← 格式从 bracket_arabic 变为 arabic，从1开始，识别为L5

[L3] 2、收入与成本
[L4] 1、费用  ← 格式相同（arabic），但从1开始，识别为L4（规则2应用）

[L4] （1）软件开发业务
[L4] （2）系统集成业务  ← 规则1优先：格式相同且数字连续，保持同级（规则2不应用）
```

#### 规则3：格式与父标题相同且连续 → 保持与父标题同级

**描述**: 如果当前标题的格式与前一个标题的格式不同，但与前一个标题的父标题格式相同，且数字连续，则当前标题与该父标题同级。

**条件**:
- 前一个标题格式不为 `None`
- 当前标题格式不为 `None`
- 格式类型不同：`current_format_type != prev_format_type`
- 存在父标题：`prev_parent_heading` 不为 `None`
- 当前格式与父标题格式相同：`current_format_type == prev_parent_format_type`
- 数字连续：`current_num == prev_parent_num + 1`

**结果**: `adjusted_level = prev_parent_level`

**示例**:
```
[L3] 2、收入与成本
  [L4] （1） 营业收入构成
  [L4] （8） 主要销售客户和主要供应商情况
[L3] 3、费用  ← 格式从 bracket_arabic 变为 arabic，但与父标题"2、收入与成本"格式相同（arabic），数字连续（2→3），保持与父标题同级L3
```

#### 规则4：格式与父标题链相同且连续 → 保持与父标题链同级

**描述**: 如果当前标题的格式与前一个标题的格式不同，但与父标题链中任何一级父标题格式相同，且数字连续，则当前标题与该父标题同级。

**条件**:
- 前一个标题格式不为 `None`
- 当前标题格式不为 `None`
- 格式类型不同：`current_format_type != prev_format_type`
- 规则3未应用
- 存在父标题链：`prev_parent_chain` 不为空
- 在父标题链中找到格式匹配且数字连续的父标题

**结果**: `adjusted_level = parent_level`（匹配的父标题层级）

**示例**:
```
[L2] 二、报告期内公司从事的主要业务
  [L3] 1、主要业务、主要产品及其用途
  [L3] 2、收入与成本
    [L4] （8） 主要销售客户和主要供应商情况
[L3] 3、费用  ← 格式与父标题链中的"2、收入与成本"格式相同（arabic）且数字连续（2→3）
```

#### 特殊规则：前一个标题格式为None且当前标题从1开始 → 子标题

**描述**: 如果前一个标题格式为 `None`（通常是"第X节"格式），且当前标题从1开始，则当前标题是前一个标题的子标题。

**条件**:
- 前一个标题格式为 `None`
- 当前标题格式不为 `None`
- 当前标题数字为1：`current_num == 1`
- 前一个标题层级为L1：`prev_level == 1`

**结果**: `adjusted_level = prev_level + 1`

**示例**:
```
[L1] 第五节 环境和社会责任
[L2] 一、重大环保问题  ← 前一个标题格式为None（第X节），当前标题从1开始，识别为子标题L2
```

### 三、规则优先级

规则按以下优先级顺序应用：

1. **规则1**（格式相同且数字连续） - **最高优先级**
2. **规则2**（从1开始） - 如果规则1未应用，则检查规则2
3. **规则3**（格式与父标题相同且连续）
4. **规则4**（格式与父标题链相同且连续）
5. **特殊规则**（前一个标题格式为None且当前标题从1开始）

**重要说明**: 
- 规则1的优先级最高，因为它是最明确的同级关系判断
- 如果规则1应用（格式相同且数字连续），会覆盖规则2的结果
- 规则2只在规则1未应用时才会生效

### 四、删除规则

如果标题不满足任何规则，且：
- 前一个标题存在
- 前一个标题层级 > 0
- 没有任何规则被应用

则该标题会被删除（层级设为0），不会出现在最终的目录结构中。这是为了确保目录结构的准确性。


## 🚀 快速开始


### Python API

```python
from chunk_by_rules import StructureGenerator, ChunkGenerator

# 第一步：生成文档结构（chunk_mode: chinese | english | simple）
structure_generator = StructureGenerator(
    document_path='document.md',
    chunk_mode='chinese'  # 中文：第X节、一、；英文：PART/ITEM全大写；简易：一级无 pattern，有序号则下一级
)
structure = structure_generator.run(output_structure='structure.json')

# 第二步：根据结构生成分块
chunk_generator = ChunkGenerator(
    document_path='document.md',
    structure_path='structure.json'
)
chunks = chunk_generator.run(
    output_json='chunks.json',
    output_md='chunks.md'
)
```

## 🔧 规则自定义

### 添加新的标题格式

编辑 `title_level_detector.py` 文件中的 `patterns` 列表：

```python
self.patterns = [
    # 添加新的格式模式
    (r'^你的正则表达式', level, '格式描述'),
    # ...
]
```

### 调整规则优先级

规则优先级在 `title_level_detector.py` 的 `detect_all_headings()` 方法中定义，按以下顺序应用：
1. 规则1（格式相同且数字连续）- 最高优先级
2. 规则2（从1开始）
3. 规则3（格式与父标题相同且连续）
4. 规则4（格式与父标题链相同且连续）
5. 特殊规则

## ⚠️ 注意事项

1. **文档格式要求**：输入文档必须是 Markdown 格式，标题使用 `#` 标记
2. **标题格式**：支持三种 chunk_mode：`chinese`（第X节、一、）、`english`（PART/ITEM 全大写）、`simple`（一级无 pattern）
3. **规则顺序很重要**：规则按优先级顺序应用，规则1的优先级最高
4. **格式识别精度**：算法会区分细微的格式差异（如空格、括号类型等），确保规则1的正确应用
5. **文件编码**：确保文档使用 UTF-8 编码
